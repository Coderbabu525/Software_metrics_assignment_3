## Overview

This repository implements an **Information Quality (IQ) Monitoring System** for source code repositories, integrating with the **Code Churn Metric** developed earlier.  
The system assesses data quality attributes for churn measurements using automated and manual checks, providing both **analytics** and **visualization**.

---

## IQ Attributes

### 1Ô∏è‚É£ Free-of-Error *(automated)*
Checks data for structural correctness and consistency using **10 different quality rules**:

| # | Check | Description |
|---|--------|--------------|
| 1 | Missing Values | Ensures dataset has no NULLs |
| 2 | Valid Dates | No unparseable or future dates |
| 3 | No Duplicates | Detects repeated entries |
| 4 | Non-negative LOC | Ensures no negative churn values |
| 5 | LOC Consistency | `modified = added + deleted` |
| 6 | Reasonable LOC Bounds | < 10,000 LOC per commit |
| 7 | Valid Commit Hash | Hash length = 8 |
| 8 | Author Validity | Valid string name |
| 9 | Module Presence | Module column not empty |
| 10 | Chronological Order | Dates increase logically |

A bar chart is produced to show which checks pass or fail.

### 2Ô∏è‚É£ Timeliness *(automated)*
Calculates the proportion of commits recorded within the last 90 days to measure **data freshness**.

### 3Ô∏è‚É£ Reputability *(manual)*
Evaluates whether commit authors belong to known or trusted organizations.  
This cannot be automatically verified ‚Äî must be manually reviewed via contributor metadata.  
**Suggested visualization:** a pie chart showing *known vs unknown* contributors.

## Manual Attribute ‚Äì Reputability
Manual verification steps:

2. Cross-check with known contributors (e.g., GitHub API or organization lists)
3. Visualize:
    - Pie chart of known vs unknown contributors
    - Optional bar chart of commits per organization
